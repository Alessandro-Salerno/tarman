USAGE:
tarman [<command>] [<options>] [<input>]

Default behaviour: show help screen.

COMMANDS:
Commands specify what tarman has to do:
* -h, --help          Show help screen
* -i, --install       Install a package
* -l, --list          List all installed packages
* -r, --remove        Remove a package
* -u, --update        Update a package if possible
* -ua, --update-all   Update all installed packages
* -ar, --add-repo     Add and update remote package repository
* -rr, --remove-repo  Remove remote package repository
* -ur, --update-repos Update contents of remote package repository

OPTIONS:
Options change the bahaviour of commands:
* -S, --super       Install a package at the system level (may require root priviledges)
* -U, --from-url    Use URL to download package
* -R, --from-repo   Use remote repository database to resolve package name
* -N, --app-name    Override application name during installation
* -X, --executable  Override executable path during installation
* -W, --working-dir Override working directory during instalaltion
* -P, --add-path    Add installed application to PATH
* -D, --add-desktop Add installed application to desktop applications

STRUCTURE OF TARMAN REPOSITORIES:
<root>/
  .tarman-repository
  package1.tarman
  package2.tarman
  package3.tarman

STRUCTURE OF `.tarman-repository` FILES:
These files are used to specify some details about the repository. These files can be found
- Inside remote tarman repositories
- Inside local tarman repositories

.tarman-repository files have one property per line. Lines are separated by \n (LF). Lines are composed of a key and a value separated by '=' as follows:
REPOSITORY_NAME=<repository name>
URL=http[s]://<path>

**THE URL IS ONLY PRESENT IN TARMAN LOCAL REPOSITORIES TO ALLOW FOR RECEPIE UPDATES**

STRUCTURE OF `.tarman` FILES:
These files specify parameters that would otherwise be specified by the user when installing a package. `.tarman` files can be found:
- Inside a tar.gz file meant to be used with tarman
- Inside package directories after installation with tarman
- Inside a tarman repository

tarman files have one property per line. Lines are separated by \n (LF). Lines are composed of a key and a value separated by '=' as follows:
URL=http[s]://<path>
APPLICATION_NAME=<application name>
EXECUTABLE=<path to executable relative to package root>
WORKING_DIRECTORY=<path of wd relative to package root>
ADD_TO_PATH={true|false}
ADD_TO_DESKTOP={true|false}
